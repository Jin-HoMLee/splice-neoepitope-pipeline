Issue #1278 / PR #1286 / fix(assemble_contigs)

The minus-strand junction that was quietly a chimera

For every minus-strand splice junction (roughly half of all junctions), the two exon arms were joined in genomic order, fabricating the splice boundary that the rest of the pipeline then translates across. The peptides it produced were nonsense - silently.

1 How a contig is built

Two flanks are cut from the reference around the junction, then bedtools getfasta -s reverse-complements each flank into transcript sense. It orients each arm - but it never reorders the pair. The concatenation order is the script's job.

upstream armA C G T AC intron (spliced out)
gene on the − strand ← downstream armGG T C C A
← On the minus strand the transcript reads 5′→3′ from the high-coordinate (downstream) arm into the low-coordinate (upstream) arm. The boxed base of each arm is the one that actually sits at the junction.
up_seq = revcomp(upstream)GTACGT dn_seq = revcomp(downstream)TGGACC
↓ This is what the script actually holds. Reverse-complementing each arm moves its junction base to the other end: the boxed base is now first in up_seq and last in dn_seq. That single fact is what makes the concatenation order decide whether the junction is real or fabricated.

old up_seq + dn_seq

GTACGT TGGACC

boundary joins T | T - the two outer edges, ~54 nt apart in the genome

A fabricated junction. The bases that meet are the far ends of the two arms, which were never adjacent in the transcript. This is what shipped for every minus-strand junction.

fix dn_seq + up_seq

TGGACC GTACGT

boundary joins C | G - the two real junction-adjacent bases

The true spliced junction: C and G are the complements of the boxed genomic bases above. Each arm was already correctly reverse-complemented; only the order needed to follow the strand.

2 Why it stayed silent

Two independent checks that should have caught it were both structurally blind to this exact shape.

CI dry-runThe Snakemake dry-run walks the DAG but never executes the script, so the concatenation never actually ran under test.
Unit fixturesEvery existing assemble_contigs fixture used strand +. The minus-strand path had zero coverage.
Forward-only translationBoth downstream consumers translate forward frames only and trust the contig is already in mRNA sense - so a flipped contig just yields wrong peptides, no error.

3 The fix

Order the arms by strand. Unknown strand . keeps genomic order (getfasta leaves those arms in forward sense and orientation cannot be inferred).

# workflow/scripts/assemble_contigs.py
- contig = (up_seq + dn_seq).upper()
+ if row["strand"] == "-":
+     contig = (dn_seq + up_seq).upper()   # transcript 5'->3': downstream arm first
+ else:
+     contig = (up_seq + dn_seq).upper()   # plus strand, and '.' fallback

4 How we know it is right

Three independent checks, each closing a gap the one before it could not.

unit · RED then GREEN

A minus-strand fixture fails on the old order and passes on the fix, plus a plus-strand guard against an unconditional swap. Confirmed red before, green after.

real bedtools E2E

Actual bedtools v2.31.1 on a chr22 minus-strand junction: the emitted contig equals the independently computed transcript-order sequence, and differs from the old order. Validates the getfasta -s assumption the stub cannot.

integration guard

A minus-strand junction carried through assembly then translation yields the transcript-correct peptide (G…A), never the chimeric one (A…G). Fails if the bug ever returns.

Impact: silently mistranslated ~half of all junctions; the fix orients once, with no downstream double-correction. Found by an adversarial bug-hunt, code-verified, and reviewed LGTM.
