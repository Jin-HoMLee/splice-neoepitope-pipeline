Proteasome-catalysed peptide splicing · Issue 681 background

Reverse cis-splicing, and why the order of two fragments can rule out RNA entirely

The spliceosome reads RNA in one direction and preserves order. A proteasome does not. That single asymmetry is the only clean way to prove a peptide was assembled after translation - and it is why some “spliced peptide” claims are structurally safe while others are indistinguishable from ordinary splice-junction neoepitopes.

One parent protein, two fragments

Everything below uses the same toy protein and the same two fragments. The only variable is the order they get ligated in - and that alone decides whether a transcript could have encoded the product.

Fragment A = TAYI (residues 3–6) Fragment B = SFVK (residues 12–15) N-terminus → C-terminus, left to right

Case 1 · An RNA splice junction (what our pipeline predicts)

Transcript-encoded

An exon-skipping event joins the exon carrying residues 3–6 directly to the exon carrying residues 12–15. The ribosome then translates that message straight through, and the resulting protein contains TAYISFVK as a contiguous stretch. The peptide is ordinary; it is the message that was novel.

Peptide presented on MHC Order preserved: 3–6 before 12–15 ✓

RNA-seq sees this. The junction is in the transcriptome, so it is sequenceable, and a sample-specific database will contain the peptide. Two independent assays can corroborate it.

Case 2 · Forward cis-splicing (the proteasome, keeping order)

Indistinguishable by sequence

Now the proteasome cuts the normal protein and ligates fragment A to fragment B, in their original order. No novel transcript exists anywhere. And yet…

Peptide presented on MHC Order preserved: 3–6 before 12–15 — identical to Case 1

This is the whole problem. The product is byte-for-byte the same peptide as the RNA-spliced one. The mass spectrometer sees one sequence and cannot tell you which history produced it. The only thing separating the two claims is whether a matching transcript exists - which reduces to an argument from absence in your RNA-seq database, and absence is a property of your database, not of the cell.

Case 3 · Reverse cis-splicing (the clean discriminator)

No transcript can encode this

Same protein. Same two fragments. But now the proteasome ligates them in reverse order - B first, then A. The C-terminal fragment ends up N-terminal in the product.

Peptide presented on MHC Order REVERSED: 12–15 before 3–6 ✗

The spliceosome joins a 5′ donor to a 3′ acceptor. It cannot run backwards, and it cannot emit an exon it has already passed. There is no transcript - annotated, novel, unassembled, or merely unsequenced - that translates to SFVKTAYI. So the RNA explanation is excluded by construction, not by the completeness of anyone's database. That is what makes this class of evidence structurally strong: a deeper RNA-seq run cannot overturn it, because no possible RNA-seq result would.

Case 4 · Trans-splicing (strong, with one caveat)

Excluded — unless a gene fusion

The two fragments come from two different proteins. No single transcript carries both sequences, so ordinary splicing is again ruled out by construction.

The caveat that must be checked: a gene fusion would place both parent sequences on one transcript, restoring an RNA explanation. So a trans-splicing claim is only as safe as the fusion screen behind it - and a fusion screen is exactly the kind of thing an RNA-seq pipeline can do and a proteomics pipeline cannot. Unlike Case 3, this exclusion still leans on the transcriptome.

What this means when you read a “spliced peptide” claim

  • Ask which topology it is, before you ask how good the FDR was. A forward cis-spliced identification is competing against an RNA explanation it cannot rule out from sequence alone. A reverse cis one is not competing with anything.
  • Exclusion arguments degrade; topology arguments do not. “It is not in our RNA-seq database” gets weaker every time someone sequences deeper. “No transcript could encode this ordering” never does.
  • The confusable case runs both ways. If a study searched spectra against UniProt without a sample-specific database containing novel junctions, then a genuine splice-junction neoepitope has nowhere canonical to map - and is a candidate to be miscalled as forward cis-spliced. Whether any published catalogue actually did this is an open question worth checking, not a finding.
  • Our peptides are the lucky ones. Splice-junction neoepitopes are transcript-encoded, so RNA-seq and mass spec are two independent assays that can corroborate each other. Proteasome-spliced peptides have no orthogonal nucleic-acid check available at all - which is precisely why that field's false-discovery fight has been so hard to settle.
